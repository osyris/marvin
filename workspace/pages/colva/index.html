PROJECT BRIEF

COLVA (CallVA)

Платформа оркестрации голосовых бизнес-процессов: входящие/исходящие звонки, аналитика, кастомные сценарии и мультипровайдерная архитектура.

Что это

COLVA — не просто “ещё один AI voice bot”. Это бизнес-слой над звонками: дашборд, логика, данные, статусы, метрики, API и процессы под конкретную цель компании.

Голосовые провайдеры (ElevenLabs, VAPI, Retell, Ultravox, кастомный LiveKit и др.) подключаются как исполнители звонка, а COLVA управляет всем процессом.

Ключевое позиционирование

“Business orchestration, not voice engine”

  • Фокус на бизнес-цели, а не на “сам факт звонка”.
  • Единый контрольный центр для inbound + outbound.
  • Структурированные результаты, а не хаос транскриптов.

Как это работает

  1. В проекте задаётся цель и сценарий (например, reminder или lead reactivation).
  2. COLVA формирует данные и переменные для звонка.
  3. Внешний провайдер совершает/принимает звонок.
  4. После звонка результат, запись, транскрипт и поля сохраняются в COLVA.
  5. Запускается post-call анализ и извлечение структурированных данных.

Сильные стороны

  • API-first: полное управление данными через API.
  • Schema per project: уникальные поля под каждый кейс.
  • Мультипроектность: 1 организация → много проектов.
  • Мультипровайдерность: разные движки в рамках одной системы.
  • Гибкие роли и доступы для команд/агентств.

Основной фокус (go-to-market)

Чтобы не быть “человеком-оркестром”, продукту нужен один острый вход на рынок:

  • Базовый фокус: напоминания о визитах (Amanati use case).
  • Дальше: реактивация лидов, опросы, collections, service flows.

Project Context Framework

Для каждого проекта фиксируются:

  • Goal: что делаем;
  • Success Criteria: как выглядит успех;
  • Negative Criteria: чего не делаем;
  • Success Metrics: как измеряем результат.
